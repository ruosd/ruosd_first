# 第二阶段企业级改进：可观测性 + 工程化

## 总体目标
让系统"看得见"内部状态，出问题时能快速定位，代码质量可量化。

## 改进清单（5 步）

| 步骤 | 模块 | 改动量 | 效果 |
|------|------|--------|------|
| 1. 结构化日志 | 全局 logger | 小 | 日志可用 grep/jq 解析 |
| 2. Prometheus 指标 | `main.py` 新增 | 小 | Grafana 可视化监控 |
| 3. 健康检查强化 | `main.py` `/health` | 小 | 一眼看出哪个依赖挂了 |
| 4. 统一错误码 | 新建 errors 模块 | 中 | 前端根据 code 而非 message 处理 |
| 5. 代码质量配置 | 新增 ruff/pyproject.toml | 小 | 自动发现潜在 bug |

## 执行顺序逻辑

- **步骤 1** 是所有可观测性的基础 — 没有结构化日志，后续的排查都靠肉眼
- **步骤 2-3** 并行建设监控的推（主动暴露指标）和拉（被动健康检查）
- **步骤 4** 是工程化基础 — 统一错误码后前后端协作更规范
- **步骤 5** 是质量兜底 — CI 之前先在本地跑通

- 第一阶段 7 步全部完成
- Docker 正常运行
