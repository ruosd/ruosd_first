# 第一阶段企业级改进：安全加固 + 可靠性

## 总体目标
在不大改架构的前提下，用最小的改动量换取最大的安全性和可靠性提升。

## 改进清单（7 步）

| 步骤 | 模块 | 改动量 | 风险 |
|------|------|--------|------|
| 1. JWT 密钥环境变量化 | `admin_router.py` + `.env` | 小 | 低 |
| 2. 密码哈希升级 MD5→bcrypt | `admin_router.py` + `user_router.py` | 中 | 中(需重置密码) |
| 3. API 请求频率限制 | 新增依赖 `slowapi` | 小 | 低 |
| 4. CORS 来源白名单 | `main.py` | 极小 | 低 |
| 5. 输入校验强化 | 所有 Pydantic 模型 | 小 | 低 |
| 6. Ollama 失效即拒绝 | `admin_router.py` + `chroma_store_service.py` | 中 | 低 |
| 7. LLM 调用自动重试 | `llm_service.py` | 小 | 低 |

## 执行顺序逻辑

为什么按这个顺序？

- **步骤 1-2** 是安全基础，没有安全的认证一切白搭
- **步骤 3-4-5** 是防御层，防止恶意请求消耗资源
- **步骤 6-7** 是可靠性，保证服务在异常情况下行为正确而非静默降级
- 每一步独立可测试，出问题可快速定位

## 前置条件

- Docker 正常运行
- 现有功能已验证可用
- 每个步骤修改后 `docker-compose up -d --build` 验证
