一、订单基本流程
================================================================================

订单处理是电商运营的核心环节，包括以下主要阶段：

【订单生命周期】
1. 订单创建 → 2. 支付确认 → 3. 商家发货 → 4. 物流运输
5. 买家收货 → 6. 交易完成 → 7. 售后处理（如有）

================================================================================
二、订单状态说明
================================================================================

【待付款】
- 定义：买家已提交订单，但尚未完成支付
- 时限：24小时内完成支付，否则自动取消
- 处理：
  · 提醒买家及时付款
  · 解答买家关于支付问题的咨询
  · 协助解决支付失败问题

【已付款/待发货】
- 定义：买家已完成支付，等待商家发货
- 时限：通常48小时内发货（特殊商品除外）
- 处理：
  · 确认支付状态
  · 准备商品
  · 安排发货

【已发货/运输中】
- 定义：商家已发货，商品正在运输途中
- 处理：
  · 提供物流单号
  · 查询物流状态
  · 解答物流相关问题
  · 处理物流异常

【已收货/交易成功】
- 定义：买家已确认收货或系统自动确认收货
- 处理：
  · 邀请买家评价
  · 处理退换货申请（如有）

【已取消】
- 定义：订单被取消
- 类型：
  · 买家取消：未付款前买家主动取消
  · 商家取消：缺货、地址无法送达等原因
  · 系统取消：超时未付款自动取消

【退款/售后中】
- 定义：涉及退款或售后处理的订单
- 处理：
  · 审核退换货申请
  · 跟踪退款进度
  · 协调商家处理

================================================================================
三、订单查询规范
================================================================================

【查询方式】
1. 按订单号查询
   - 输入格式：订单号（纯数字或字母组合）
   - 示例：2026060812345678

2. 按时间查询
   - 支持按日期范围筛选
   - 默认显示最近一个月的订单

3. 按状态查询
   - 待付款/已付款/已发货/已完成/已取消

【查询结果展示】
- 订单基本信息（订单号、下单时间、支付方式）
- 商品信息（商品名称、数量、价格）
- 物流信息（物流公司、单号、当前状态）
- 金额信息（商品金额、运费、优惠、实付金额）

================================================================================
四、物流信息规范
================================================================================

【常用物流公司】
- 顺丰速运
- 中通快递
- 圆通速递
- 韵达快递
- 申通快递
- 京东物流
- EMS邮政

【物流状态说明】
- 已发货：商家已发货，商品已发出
- 运输中：商品正在运输途中
- 派送中：商品到达目的站点，正在派送
- 已签收：商品已被签收
- 异常：物流出现异常（延迟、丢失等）

【物流异常处理】
1. 延误：联系物流公司核实，预计送达时间
2. 丢失：启动理赔流程，必要时重新发货
3. 拒收：与买家沟通，了解原因并处理
4. 退回：跟进退回物流，协助重新发货

================================================================================
五、订单金额计算
================================================================================

【订单金额组成】
商品总价 = Σ(商品单价 × 数量)

订单总额 = 商品总价 + 运费 - 优惠金额

实付金额 = 订单总额 - 使用积分/优惠券

【常见优惠类型】
- 满减优惠：满XX元减XX元
- 折扣优惠：X折销售
- 优惠券：满减券/无门槛券
- 积分抵扣：100积分抵1元
- 活动优惠：限时特价、节日活动

================================================================================
六、订单修改规则
================================================================================

【可修改项】
1. 收货地址（待发货状态）
2. 收货人电话（待发货状态）
3. 备注信息
4. 发票信息

【不可修改项】
1. 商品规格/数量
2. 支付方式
3. 已发货订单的收货信息

【修改时限】
- 收货信息：订单发货前可修改
- 商品信息：支付前可修改
- 发票信息：交易完成前可修改

================================================================================
七、订单异常处理
================================================================================

【常见异常类型】

1. 恶意下单
   - 特征：同一IP大量下单、虚假信息
   - 处理：核实后取消订单，标记用户

2. 重复下单
   - 特征：同一用户短时间内多次下单相同商品
   - 处理：联系买家确认，取消重复订单

3. 地址错误
   - 特征：收货地址不详细、无法配送
   - 处理：联系买家确认正确地址

4. 商品缺货
   - 特征：库存不足，无法按期发货
   - 处理：与买家协商换货或退款

5. 超时未付款
   - 定义：下单后24小时未付款
   - 处理：系统自动取消订单

================================================================================
八、订单数据分析
================================================================================

【核心指标】
1. 订单量 - 单位时间内的订单总数
2. 订单金额 - 总销售额
3. 客单价 = 销售额 / 订单数
4. 转化率 = 成单数 / 访客数
5. 取消率 = 取消订单数 / 总订单数

【分析维度】
- 按时间：日/周/月/年
- 按商品：单品/品类
- 按用户：新用户/老用户
- 按渠道：PC端/移动端
